# 演示剧本

## 场景一：写邮件

**说**: "帮我写一封请假邮件，昨天就有点发烧今天还没好，跟张经理请个病假"

**期望输出**:
```
Subject: 病假申请

张经理您好，

我从昨天开始发烧，今天仍未好转，特申请病假一天。
相关工作我会尽量远程处理，如有紧急事项请随时联系。

祝好，
[您的名字]
```

**体现创新**: 语音输入法不是逐字转录——LLM 理解这是请假需求，自动补全称呼、正文、落款。

## 场景二：会议记录→结构化总结

**说**: "今天例会讨论了三个事情。第一，新版本下周五上线，王磊负责后端接口，李娜负责前端页面。第二，上次的 bug 修复需要重新测试，张伟来负责。第三，下周要准备季度汇报的 PPT，我来写。"

**期望输出**:
```
- 新版本下周五上线
  - 王磊：后端接口
  - 李娜：前端页面
- Bug 修复重新测试
  - 张伟负责
- 季度汇报 PPT
  - 我来写
```

**体现创新**: 散乱的口语自动整理为结构化清单，LLM 理解层级关系。

## 场景三：长段口述→通顺文章

**说**: "这个项目的核心思路就是把语音识别和 LLM 结合起来，传统的语音输入法只是把语音转成文字，但是我们在这个基础上加了一个 LLM 的处理层，它可以理解用户到底想表达什么，然后自动地整理成合适的格式，比如你想写邮件它就帮你写成邮件，你在开会做记录它就帮你总结成要点，这样就比普通的语音输入法要聪明很多。"

**期望输出**: 一段通顺、有标点、分段落的文字，去除口语重复。

**体现创新**: 从"语音转录"升级为"语音理解输出"。

## 场景四：代码注释

**说**: "写个注释，这个函数检查输入的手机号是不是有效的，如果是返回true，不是返回 false，输入是字符串"

**期望输出**:
```
// 检查输入的手机号是否为有效格式
// 参数: phone - 待验证的手机号字符串
// 返回: true 表示格式有效，false 表示格式无效
```

**体现创新**: 口语描述→规范的代码注释，知道函数签名和返回值。
